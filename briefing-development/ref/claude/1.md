## KWP2000 (Keyword Protocol 2000) — ISO 14230

KWP2000 adalah protokol diagnostik kendaraan generasi kedua yang menggantikan ISO 9141. Dipakai luas di kendaraan Eropa dan Asia tahun 1996–2008, sebelum OBD-II wajib pakai CAN (ISO 15765).

---

### Physical Layer — K-Line

Komunikasi lewat satu kabel K-Line (kadang ada L-Line untuk init). Karakteristiknya:

- **Baudrate**: 10.4 kbaud default (bisa dinegosiasi)
- **Logic level**: idle = HIGH (12V), dominant = LOW (0V)
- **Inisiasi**: ada dua cara — 5-baud init (lambat, kompatibel semua ECU) dan fast init (25ms LOW pulse)
- **ECU address standar**: `0x33` untuk tester, ECU engine biasanya `0x10`-`0x1F`

---

### Frame Format (ISO 14230-2)

```
[FMT] [TGT] [SRC] [LEN?] [DATA...] [CS]
```

- `FMT` (Format byte): bit 7-6 = addressing mode, bit 5-4 = reserved, bit 3-0 = length (0 = pakai byte LEN terpisah)
- `TGT` dan `SRC` hanya ada di **functional addressing**
- `CS` = checksum XOR atau sum semua byte

Contoh request `ReadDataByLocalIdentifier` (SID `0x21`):

```
80 10 F1 02 21 0F A3
│  │  │  │  │  │  └─ checksum
│  │  │  │  │  └──── local ID yang diminta
│  │  │  │  └─────── service ID
│  │  │  └────────── length = 2 byte data
│  │  └───────────── source = 0xF1 (tester)
│  └──────────────── target = 0x10 (ECU)
└─────────────────── format = 0x80 (with addr, no length in fmt)
```

---

### Diagnostic Services (SID)

| SID | Nama | Fungsi |
|-----|------|--------|
| `0x10` | StartDiagnosticSession | Buka sesi (default/programming/extended) |
| `0x11` | ECUReset | Reset ECU (soft/hard/power) |
| `0x14` | ClearDiagnosticInformation | Hapus DTC |
| `0x17` | ReadStatusOfDTCByStatusMask | Baca DTC by status |
| `0x18` | ReadDTCByStatus | Baca semua DTC + status |
| `0x21` | ReadDataByLocalIdentifier | Baca data live (RPM, coolant temp, dll) |
| `0x22` | ReadDataByCommonIdentifier | Baca by DID (16-bit) |
| `0x27` | SecurityAccess | Seed/key unlock untuk programming |
| `0x2C` | DynamicallyDefineLocalIdentifier | Definisi data custom |
| `0x34` | RequestDownload | Flash programming start |
| `0x36` | TransferData | Kirim data firmware |
| `0x3E` | TesterPresent | Keep-alive — cegah timeout sesi |

---

### Security Access (SID 0x27) — Alur Seed/Key

```
Tester → ECU:  27 01          (request seed, level 1)
ECU → Tester:  67 01 A3 F2    (seed = 0xA3F2)
Tester → ECU:  27 02 C5 1D    (key = hasil kalkulasi dari seed)
ECU → Tester:  67 02          (access granted)
```

Algoritma key berbeda tiap OEM — ini yang bikin ECU programming tidak sembarangan bisa dilakukan.

### Relevansi ke ECU Simulator-mu

Di codebase kamu, ini mapping-nya:

| Komponen simulator | Protokol |
|---|---|
| `packages/protocols/kwp2000` | Frame parser + service router |
| `packages/services/security-engine` | Seed/key algorithm |
| `packages/services/session-fsm` | State machine: default → extended → programming |
| `packages/services/dtc-engine` | Penyimpanan + query DTC per SAE J2012 |
| `packages/transport` | K-Line serial (hardware) atau TCP (dev) |

Yang paling kritis untuk diimplementasi duluan sebelum yang lain adalah **session FSM** — karena hampir semua service bergantung pada status sesi aktif. Tanpa itu, simulator tidak bisa enforce behavior yang benar.
