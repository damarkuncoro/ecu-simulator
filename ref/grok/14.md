**Detailed Mathematics for Game 1 (HKDF Proof)**

Here is the precise mathematical treatment of the transition from **Game 0** (real HKDF) to **Game 1** (random PRK).

### Game Definitions

**Game 0 (Real HKDF)**:
- Challenger samples master secret $IKM \leftarrow \{0,1\}^\lambda$ (or adversary supplies it).
- On query $(salt, info, len)$:
  $$
  PRK \gets \mathsf{HMAC}(salt, IKM)
  $$
  $$
  output \gets \mathsf{Expand}(PRK, info, len)
  $$
- Return $output$.

**Game 1 (Random PRK)**:
- On query $(salt, info, len)$:
  $$
  PRK \xleftarrow{\$ } \{0,1\}^{n} \quad \text{(uniform random, } n = \text{hash output length)}
  $$
  $$
  output \gets \mathsf{Expand}(PRK, info, len)
  $$
- Return $output$.

---

### Advantage Difference (Reduction)

**Lemma 1**:
$$
|\Pr[\mathcal{A}^{\text{Game0}} = 1] - \Pr[\mathcal{A}^{\text{Game1}} = 1]| \leq \mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}(\mathcal{B})
$$

Where $\mathcal{B}$ is a PRF adversary constructed from $\mathcal{A}$.

### Detailed Reduction (How $\mathcal{B}$ Works)

**$\mathcal{B}$** (PRF distinguisher against HMAC):

1. $\mathcal{B}$ receives a PRF challenge oracle $\mathcal{O}$ (either real $\mathsf{HMAC}(k,\cdot)$ or random function $R(\cdot)$).
2. $\mathcal{B}$ picks a random $IKM$ (or uses the one from $\mathcal{A}$ if adaptive).
3. When $\mathcal{A}$ makes an HKDF query $(salt_i, info_i, len_i)$:
   - $\mathcal{B}$ queries its oracle: $PRK_i \gets \mathcal{O}(salt_i)$   ← This is the critical step.
   - Then computes $output_i \gets \mathsf{Expand}(PRK_i, info_i, len_i)$ using the normal Expand algorithm.
   - Returns $output_i$ to $\mathcal{A}$.
4. When $\mathcal{A}$ outputs its guess $b'$, $\mathcal{B}$ outputs the same bit.

### Analysis

- **If $\mathcal{O}$ is real HMAC** (with secret key = $IKM$):  
  The view of $\mathcal{A}$ is **identical to Game 0**.

- **If $\mathcal{O}$ returns random values**:  
  The view of $\mathcal{A}$ is **identical to Game 1**.

Therefore:
$$
\mathsf{Adv}^{\text{Game0 vs Game1}}(\mathcal{A}) = \mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}(\mathcal{B})
$$

This is a **tight reduction** — the advantage is exactly preserved.

### Why This Works

- The Extract phase is **exactly** one call to HMAC(salt, IKM).
- The security of the entire HKDF reduces directly to the PRF security of this single call.
- The Expand phase is handled in later games (Game 1 → Game 2, etc.).

### Concrete Security Interpretation

If there is an adversary $\mathcal{A}$ that breaks HKDF with advantage $\epsilon$ using $q$ queries, then there exists an adversary $\mathcal{B}$ that breaks HMAC-PRF security with advantage roughly $\epsilon$ (making at most $q$ queries to its oracle).

This is why HKDF is considered **cryptographically strong** — its security rests directly on the well-studied security of HMAC.
