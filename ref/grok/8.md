**Key Derivation Techniques for Security Access (0x27)**

**Key Derivation** is a critical part of modern Security Access implementations. Instead of using a single static secret key, ECUs often derive per-session or per-level keys from a **master secret** + inputs like seed, security level, ECU ID, or a nonce. This improves security against key reuse and replay attacks.

### Why Use Key Derivation?

- Avoids hard-coding many static keys.
- Allows different access levels to have cryptographically isolated keys.
- Makes reverse engineering harder.
- Supports forward secrecy in some designs.

### Common Key Derivation Methods

Here are practical techniques, ordered from simple to advanced, with **TypeScript** examples suitable for your `security-engine`.

#### 1. Simple Concat + Hash (Very Common)

```ts
import { createHash } from 'crypto';

function deriveKeySimple(
  masterSecret: Buffer,
  seed: Buffer,
  level: number,
  outputLength: number = 16
): Buffer {
  const data = Buffer.concat([
    masterSecret,
    seed,
    Buffer.from([level])
  ]);

  const hash = createHash('sha256')
    .update(data)
    .digest();

  return hash.subarray(0, outputLength);   // Truncate to AES-128 key size
}
```

#### 2. HKDF (HMAC-based Key Derivation Function) — Recommended

HKDF (RFC 5869) is the industry standard for this use case.

```ts
import { createHmac } from 'crypto';

function hkdf(
  masterSecret: Buffer,
  seed: Buffer,
  level: number,
  salt: Buffer = Buffer.alloc(0),
  info: Buffer = Buffer.from('diag-security'),
  outputLength: number = 16
): Buffer {
  // Extract
  const prk = createHmac('sha256', salt)
    .update(masterSecret)
    .digest();

  // Expand
  const infoBuffer = Buffer.concat([
    info,
    Buffer.from([level]),
    seed
  ]);

  const t = createHmac('sha256', prk)
    .update(infoBuffer)
    .digest();

  return t.subarray(0, outputLength);
}
```

#### 3. AES-Based Key Derivation (AES-KDF)

```ts
import { createCipheriv } from 'crypto';

function aesKDF(
  masterKey: Buffer,   // AES key
  seed: Buffer,
  level: number,
  iterations: number = 1000
): Buffer {
  let key = Buffer.concat([masterKey, seed, Buffer.from([level])]);
  
  for (let i = 0; i < iterations; i++) {
    const cipher = createCipheriv('aes-128-ecb', masterKey, null as any);
    key = Buffer.concat([cipher.update(key), cipher.final()]);
  }
  
  return key.subarray(0, 16);   // AES-128 key
}
```

#### 4. Full Example Class for Your Simulator

```ts
class DerivedSecurityAlgorithm {
  private masterSecret: Buffer;

  constructor(masterSecretHex: string) {
    this.masterSecret = Buffer.from(masterSecretHex, 'hex');
  }

  calculateKey(seed: Buffer, level: number): Buffer {
    // Choose your preferred derivation
    return this.hkdfDerive(seed, level);
  }

  private hkdfDerive(seed: Buffer, level: number): Buffer {
    const salt = Buffer.from('ecu-diagnostic-salt', 'utf8');
    const info = Buffer.from('kwp2000-security', 'utf8');

    // Simple HKDF-like
    const prk = createHmac('sha256', salt)
      .update(this.masterSecret)
      .digest();

    const expand = createHmac('sha256', prk)
      .update(Buffer.concat([info, Buffer.from([level]), seed]))
      .digest();

    return expand.subarray(0, 16);   // AES-128 key
  }
}
```

### Usage in Security Access Flow

```ts
// In your KWP2000 service handler
async handleSecurityAccess(request: Buffer, level: number) {
  if (isRequestSeed(request)) {
    const seed = this.generateSecureSeed(8);           // 8 bytes example
    this.currentSeed = seed;
    return buildPositiveResponse(0x67, level, seed);
  } 
  
  else if (isSendKey(request)) {
    const receivedKey = extractKey(request);
    const expectedKey = this.securityAlgo.calculateKey(this.currentSeed!, level);
    
    if (receivedKey.equals(expectedKey)) {
      this.unlockLevel(level);
      return buildPositiveResponse(0x67, level);
    } else {
      return negativeResponse(0x27, 0x35); // invalidKey
    }
  }
}
```

### Best Practices for Your ECU Simulator

- **Configurable per OEM** — Different derivation methods/params per virtual ECU.
- **Salt & Info** — Make them configurable (ECU-specific constants).
- **Output Length** — Match the expected key length for the security level.
- **Randomness** — Use cryptographically secure seed generation (`crypto.randomBytes`).
- **Replay Protection** — Store used seeds temporarily and reject repeats.
- **Multiple Levels** — Derive different keys for level 1 (diagnostics) vs level 9 (programming).
