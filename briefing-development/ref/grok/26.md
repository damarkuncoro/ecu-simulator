**HKDF Benchmark: Different Output Lengths**

Here are the results for full HKDF (Extract + Expand) with varying output lengths.

### Benchmark Results (2,000 operations per test)

| Output Length | SHA-256 Time | SHA-512 Time | SHA-512 Speed | Notes |
|---------------|--------------|--------------|---------------|-------|
| 16 bytes      | 0.0086 s     | 0.0119 s     | 1.39x slower  | Typical for AES-128 |
| 32 bytes      | 0.0084 s     | 0.0118 s     | 1.40x slower  | Typical for AES-256 |
| 64 bytes      | 0.0128 s     | 0.0117 s     | **1.09x faster** | SHA-512 advantage starts |
| 128 bytes     | 0.0212 s     | 0.0179 s     | **1.18x faster** | SHA-512 clearly wins |

### Key Insights

- For **short outputs** (16–32 bytes) — **SHA-256 is faster**.
- For **longer outputs** (≥64 bytes) — **SHA-512 becomes faster** because it processes data in larger blocks and requires fewer iterations in the Expand phase.

### Practical Recommendation for Your Simulator

| Security Level / Use Case       | Recommended Hash | Output Length | Reason |
|---------------------------------|------------------|---------------|--------|
| Standard Diagnostic Access      | SHA-256          | 16–32 bytes   | Best performance |
| High Security (Flashing/Coding) | SHA-512          | 32–64 bytes   | Stronger margin |
| Session / Traffic Keys          | SHA-256          | 32 bytes      | Sweet spot |
| MAC + Encryption Keys           | SHA-512          | 64 bytes      | Future-proof |
