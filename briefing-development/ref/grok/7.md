**AES-Based Security Access Algorithms (0x27)**

Modern ECUs (especially post-2015+) increasingly use **AES** (usually AES-128 or AES-256) for seed-key calculations. This is much stronger than simple XOR or linear methods.

### Common AES Approaches in ECUs

1. **AES-ECB** — Direct encrypt of seed (or padded seed) with secret key.
2. **AES-CBC** — With fixed or derived IV.
3. **AES-CMAC** — For authentication (more secure).
4. **AES Encrypt + Post-Processing** — Truncate or XOR the output.

**Important**: The **secret key** is stored in the ECU (and known by authorized diagnostic tools). It is never transmitted.

### TypeScript Examples (Using Web Crypto API)

Node.js / modern browsers support this natively.

#### Basic AES-128 ECB Style (Most Straightforward)

```ts
async function calculateAESKey(
  seed: Buffer,
  secretKey: Buffer,        // 16 bytes for AES-128, 32 for AES-256
  level: number = 1
): Promise<Buffer> {
  if (secretKey.length !== 16 && secretKey.length !== 32) {
    throw new Error("Secret key must be 16 or 32 bytes");
  }

  // Import key
  const cryptoKey = await crypto.subtle.importKey(
    "raw",
    secretKey,
    { name: "AES-ECB" },   // Note: ECB is not in Web Crypto by default in some envs
    false,
    ["encrypt"]
  );

  // Pad seed to block size (16 bytes) if needed
  const paddedSeed = padPKCS7(seed, 16);

  const encrypted = await crypto.subtle.encrypt(
    { name: "AES-ECB" },
    cryptoKey,
    paddedSeed
  );

  // Take first N bytes matching seed length (common practice)
  return Buffer.from(encrypted).subarray(0, seed.length);
}

function padPKCS7(data: Buffer, blockSize: number): Buffer {
  const padding = blockSize - (data.length % blockSize);
  const padded = Buffer.alloc(data.length + padding);
  data.copy(padded);
  padded.fill(padding, data.length);
  return padded;
}
```

**Note**: Pure Web Crypto does **not** support ECB directly in all environments (for security reasons). Use `crypto` module in Node.js or a library like `crypto-js` / `node-forge`.

#### Using `node:crypto` (Recommended for Your Project)

```ts
import { createCipheriv, createDecipheriv } from 'crypto';

class AESSecurityAlgorithm {
  private secretKey: Buffer;
  private algorithm: string = 'aes-128-ecb';   // or aes-256-cbc

  constructor(secretKeyHex: string) {
    this.secretKey = Buffer.from(secretKeyHex, 'hex');
  }

  calculateKey(seed: Buffer, level: number): Buffer {
    // For ECB
    const cipher = createCipheriv(this.algorithm, this.secretKey, null as any);
    let encrypted = cipher.update(seed);
    encrypted = Buffer.concat([encrypted, cipher.final()]);

    // Common: return same length as seed
    return encrypted.subarray(0, seed.length);
  }
}

// Usage
const algo = new AESSecurityAlgorithm('0123456789ABCDEFFEDCBA9876543210'); // 16-byte key
const seed = Buffer.from('A1B2C3D4', 'hex');
const key = algo.calculateKey(seed, 1);
console.log('Key:', key.toString('hex'));
```

### AES-CBC Example (With IV)

```ts
calculateKeyCBC(seed: Buffer, level: number): Buffer {
  const iv = Buffer.alloc(16, 0);           // Fixed IV or derived from level/seed
  const cipher = createCipheriv('aes-128-cbc', this.secretKey, iv);
  
  let encrypted = cipher.update(seed);
  encrypted = Buffer.concat([encrypted, cipher.final()]);

  return encrypted.subarray(0, seed.length);
}
```

### AES-CMAC (More Secure, Recommended for New Designs)

You can use `node-forge` or implement CMAC.

```ts
// Pseudo-code using a library
import { CMAC } from 'some-cmac-lib';

calculateCMACKey(seed: Buffer, secretKey: Buffer): Buffer {
  const cmac = new CMAC(secretKey);
  cmac.update(seed);
  const mac = cmac.final();           // 16 bytes
  return mac.subarray(0, seed.length); // Truncate if needed
}
```

### Integration Recommendations for Your Simulator

1. **Configurable per OEM/Profile**:
   ```ts
   securityEngine.register('modern-bosch', 1, new AESSecurityAlgorithm(keyHex));
   ```

2. **Multiple Variants**:
   - Store different keys per security level.
   - Support key derivation (e.g., HKDF from master key + level).

3. **Seed Handling**:
   - Generate cryptographically random seeds.
   - Support variable seed lengths (4, 8, 16 bytes common).

4. **Performance & Testing**:
   - Cache imported keys if using Web Crypto.
   - Provide test vectors (known seed → expected key) in your test suite.

**Security Note**: In a real ECU, the secret key is protected (e.g., in secure memory). In your simulator, load it from `.env` or config files (never commit real keys).
