**Detailed Mathematics for Game 2 (HKDF Proof)**

Here is the precise treatment of the transition from **Game 1** (random PRK) to **Game 2** (random outputs in Expand phase).

### Game Definitions (Reminder)

**Game 1** (Random PRK):
- On query $(salt, info, len)$:
  $$
  PRK \xleftarrow{\$ } \{0,1\}^n \quad \text{(uniform random)}
  $$
  $$
  output \gets \mathsf{Expand}(PRK, info, len)
  $$

**Game 2** (Ideal Expand):
- On query $(salt, info, len)$:
  $$
  PRK \xleftarrow{\$ } \{0,1\}^n
  $$
  $$
  output \xleftarrow{\$ } \{0,1\}^{len} \quad \text{(pure uniform random string)}
  $$

No Expand computation is performed — the output is directly random.

---

### Advantage Difference (Reduction)

**Lemma 2**:
$$
|\Pr[\mathcal{A}^{\text{Game1}} = 1] - \Pr[\mathcal{A}^{\text{Game2}} = 1]| \leq q \cdot \mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}(\mathcal{C})
$$

Where $q$ is the number of HKDF queries made by $\mathcal{A}$, and $\mathcal{C}$ is a PRF adversary.

### Detailed Reduction Construction

**$\mathcal{C}$** (PRF distinguisher for the Expand phase):

1. $\mathcal{C}$ receives a PRF challenge oracle $\mathcal{O}_k(\cdot)$ (either real $\mathsf{HMAC}(k, \cdot)$ with secret $k=PRK$, or a random function $R(\cdot)$).

2. $\mathcal{C}$ simulates **Game 1** for $\mathcal{A}$:
   - For each HKDF query $(salt_i, info_i, len_i)$ from $\mathcal{A}$:
     - Sample fresh random $PRK_i \xleftarrow{\$ } \{0,1\}^n$ (as in Game 1).
     - Simulate the Expand chain **using the oracle $\mathcal{O}$**:
       $$
       T_0 = \epsilon \quad \text{(empty)}
       $$
       For $j = 1$ to enough blocks:
       $$
       T_j = \mathcal{O}(T_{j-1} \, || \, info_i \, || \, 0x0j)
       $$
     - Concatenate the $T_j$ blocks and truncate to length $len_i$ to form $output_i$.
     - Return $output_i$ to $\mathcal{A}$.

3. When $\mathcal{A}$ outputs its guess $b'$, $\mathcal{C}$ outputs the same bit.

### Why the Reduction Holds

- **When $\mathcal{O}$ is real HMAC** (with key = PRK):  
  The simulation is **statistically identical to Game 1**.

- **When $\mathcal{O}$ is a random function**:  
  Each $T_j$ is random, and thus the final $output$ is random → **identical to Game 2**.

Because each query uses a fresh PRK (or the oracle is called with different inputs due to the counter), the reduction is tight up to a factor of $q$ (number of queries).

---

### Full Chain Summary (All Games)

- **Game 0 → Game 1**: Advantage bounded by $\mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}$ (Extract reduction)
- **Game 1 → Game 2**: Advantage bounded by $q \cdot \mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}$ (Expand reduction)
- **Game 2**: Advantage = 0 (pure random vs random)

**Overall Bound**:
$$
\text{Adv}^{\text{HKDF}}(\mathcal{A}) \leq \mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}(\mathcal{B}) + q \cdot \mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}(\mathcal{C})
$$

This is an excellent concrete security bound for practical parameters.
