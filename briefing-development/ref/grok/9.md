**HKDF (HMAC-based Extract-and-Expand Key Derivation Function)** — Detailed Breakdown

**HKDF** (RFC 5869) is one of the best and most widely recommended key derivation methods for Security Access in modern ECUs. It consists of two distinct phases:

### 1. Extract Phase
Turns the (possibly low-entropy) master secret into a high-entropy **pseudorandom key (PRK)** using a salt.

**Purpose**: "Extract" uniform randomness from the input keying material.

**Formula**:
```ts
PRK = HMAC-Hash(salt, IKM)
```
- `IKM` = Input Keying Material (your master secret)
- `salt` = Optional random or fixed value (improves security)
- Output: `PRK` (usually same length as hash output, e.g. 32 bytes for SHA-256)

### 2. Expand Phase
Expands the PRK into one or more output keys of desired length using an `info` string (context).

**Formula** (for one block):
```ts
T(1) = HMAC-Hash(PRK, info || 0x01)
Output = T(1)  (truncated to desired length)
```

For longer outputs (multiple blocks):
```ts
T(1) = HMAC-Hash(PRK, info || 0x01)
T(2) = HMAC-Hash(PRK, T(1) || info || 0x02)
T(3) = HMAC-Hash(PRK, T(2) || info || 0x03)
...
```

### Complete HKDF Implementation (TypeScript)

```ts
import { createHmac } from 'crypto';

class HKDF {
  private hash: string = 'sha256';   // or sha512
  private hashLen: number = 32;      // SHA-256 output size

  /**
   * Full HKDF Extract + Expand
   */
  derive(
    ikm: Buffer,                    // Master secret / Input Keying Material
    salt: Buffer = Buffer.alloc(0),
    info: Buffer = Buffer.alloc(0),
    outputLength: number = 16,      // Usually 16 (AES-128) or 32 (AES-256)
    level: number = 1,
    seed: Buffer = Buffer.alloc(0)
  ): Buffer {
    const prk = this.extract(ikm, salt);
    return this.expand(prk, info, outputLength, level, seed);
  }

  private extract(ikm: Buffer, salt: Buffer): Buffer {
    return createHmac(this.hash, salt.length ? salt : Buffer.alloc(this.hashLen, 0))
      .update(ikm)
      .digest();
  }

  private expand(
    prk: Buffer,
    info: Buffer,
    outputLength: number,
    level: number,
    seed: Buffer
  ): Buffer {
    const contextInfo = Buffer.concat([
      info,
      Buffer.from([level]),
      seed
    ]);

    let result = Buffer.alloc(0);
    let t = Buffer.alloc(0);
    let counter = 1;

    while (result.length < outputLength) {
      const hmac = createHmac(this.hash, prk);
      
      if (t.length > 0) hmac.update(t);
      hmac.update(contextInfo);
      hmac.update(Buffer.from([counter]));

      t = hmac.digest();
      result = Buffer.concat([result, t]);
      counter++;
    }

    return result.subarray(0, outputLength);
  }
}
```

### Practical Usage in Security Access

```ts
const hkdf = new HKDF();

function calculateSecurityKey(
  masterSecret: Buffer,
  seed: Buffer,
  level: number
): Buffer {
  return hkdf.derive(
    masterSecret,
    Buffer.from('ecu-diagnostic-salt'),   // Fixed per ECU family
    Buffer.from('kwp2000-uds-security'), // Context string
    16,                                   // 16 bytes = AES-128 key
    level,
    seed
  );
}
```

### Recommended Parameters

- **Hash** — SHA-256 (best balance) or SHA-512.
- **Salt** — ECU-specific constant (can be public).
- **Info** — Should include purpose ("security-access-level-1", "flash-programming", etc.).
- **Output Length** — Match the algorithm you will use (16 bytes for AES-128, 32 for AES-256).
- **Level & Seed** — Feed both into the `info` context to make keys unique per request.
