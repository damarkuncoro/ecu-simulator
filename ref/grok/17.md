**Proof of the Extract Phase Reduction (Game 0 → Game 1)**

Here is a clean, self-contained formal proof.

### Games (Focused on Extract)

**Game 0 – Real HKDF**:
- Challenger has fixed master secret $IKM$.
- On adversary query $(salt, info, len)$:
  $$
  PRK \ :=\ \mathsf{HMAC}(salt,\ IKM)
  $$
  $$
  output \ :=\ \mathsf{Expand}(PRK,\ info,\ len)
  $$
- Return $output$.

**Game 1 – Random PRK**:
- On query $(salt, info, len)$:
  $$
  PRK \ \xleftarrow{\$}\ \{0,1\}^n \quad \text{(uniform random)}
  $$
  $$
  output \ :=\ \mathsf{Expand}(PRK,\ info,\ len)
  $$

---

### Theorem

$$
\left| \Pr[\mathcal{A}\ \text{wins Game 0}] - \Pr[\mathcal{A}\ \text{wins Game 1}] \right| \ \leq\ \mathsf{Adv}^{\mathsf{PRF}}_{\mathsf{HMAC}}(\mathcal{B})
$$

for an efficient PRF adversary $\mathcal{B}$ whose resources are essentially the same as $\mathcal{A}$’s.

### Reduction: Construction of $\mathcal{B}$

**$\mathcal{B}$** is a PRF distinguisher against HMAC:

1. $\mathcal{B}$ receives a PRF challenge oracle $\mathcal{O}$ (either real $\mathsf{HMAC}(k,\cdot)$ with hidden key $k$, or a truly random function).

2. $\mathcal{B}$ runs $\mathcal{A}$ as a subroutine:
   - Whenever $\mathcal{A}$ issues an HKDF query $(salt_i, info_i, len_i)$:
     - $\mathcal{B}$ makes **one query** to its oracle:
       $$
       PRK_i \ \gets\ \mathcal{O}(salt_i)
       $$
     - Then runs the normal (deterministic) Expand algorithm:
       $$
       output_i \ \gets\ \mathsf{Expand}(PRK_i,\ info_i,\ len_i)
       $$
     - Gives $output_i$ back to $\mathcal{A}$.

3. When $\mathcal{A}$ outputs its final guess $b'$, $\mathcal{B}$ outputs the same $b'$ as its answer to the PRF challenger.

### Correctness

- **Real HMAC case** ($\mathcal{O} = \mathsf{HMAC}(k,\cdot)$):  
  $PRK_i = \mathsf{HMAC}(salt_i, IKM)$ exactly.  
  $\mathcal{A}$ sees **Game 0**.

- **Random function case** ($\mathcal{O} = R(\cdot)$):  
  Each $PRK_i$ is uniformly random and independent of $salt_i$.  
  $\mathcal{A}$ sees **Game 1**.

Therefore, $\mathcal{B}$ wins its PRF game with exactly the same advantage that $\mathcal{A}$ has in distinguishing Game 0 from Game 1.

The reduction is **tight** (no significant loss factor) because the Extract phase consists of **exactly one HMAC call** per query.

---

This reduction shows that the security of the Extract phase rests directly on the PRF security of HMAC.
